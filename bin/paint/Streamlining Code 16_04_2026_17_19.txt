  ---
  Das Kernproblem

  Fast jede Methode greift direkt auf private-Felder der äußeren Klasse zu (workingImage, canvases, secWin,
  paintToolbar, etc.). Deshalb sind Inner-Class-Methoden so gefährlich zu verschieben — Java-Sichtbarkeitsregeln machen
  das leichter als in anderen Sprachen, aber man muss private → package-private ändern.

  ---
  Empfohlene Strategie: Inkrementell, risikogeordnet

  Schritt 1 – Kein Risiko: Static Inner Classes heraus (sofort machbar)

  TransferableImage (Z. 5053) ist eine private static class ohne Zugriff auf äußere Felder → einfach als
  TransferableImage.java in package paint auslagern.

  // Vorher (in SelectiveAlphaEditor):
  private static class TransferableImage implements Transferable { ... }

  // Nachher: eigene Datei paint/TransferableImage.java
  class TransferableImage implements Transferable { ... }

  ---
  Schritt 2 – Geringes Risiko: SecondaryPanel auslagern

  SecondaryPanel (Z. 5078–5303) ist eine private class mit Outer-Referenz. Pattern:

  // paint/SecondaryPanel.java
  class SecondaryPanel extends JPanel {
      private final SelectiveAlphaEditor editor; // statt implizitem outer-this

      SecondaryPanel(SelectiveAlphaEditor editor) {
          this.editor = editor;
          // alle Zugriffe: editor.canvases statt canvases
      }
  }

  Alle Zugriffe auf äußere Felder werden zu editor.fieldName. Die betroffenen Felder müssen von private auf
  package-private (kein Modifier) gesetzt werden.

  ---
  Schritt 3 – Mittleres Risiko: Controller-Klassen mit Editor-Referenz

  Das bewährteste Pattern für eng gekoppelte JFrame-Klassen: Controller-Objekte, die SelectiveAlphaEditor als
  Konstruktorparameter bekommen.

  Kandidaten:

  KeyboardShortcutManager.java    ← setupKeyBindings() + alle AbstractAction-Definitionen
  ClipboardController.java        ← doCopy/doPaste/doCut (nutzt floatSelectionState)
  FileLoadController.java         ← loadFile(), loadImage(), saveFile() etc.
  SecondaryWindowController.java  ← initSecondaryWindow(), openSecondaryWindow(), Timer-Setup

  Beispiel-Pattern:

  class KeyboardShortcutManager {
      private final SelectiveAlphaEditor ed;

      KeyboardShortcutManager(SelectiveAlphaEditor ed) {
          this.ed = ed;
      }

      void setup() {
          // bisheriger Inhalt von setupKeyBindings()
          // ed.doUndo() statt doUndo()
          // ed.ci() statt ci()
      }
  }

  In SelectiveAlphaEditor:
  private final KeyboardShortcutManager keyboardManager = new KeyboardShortcutManager(this);
  // in initializeUI():
  keyboardManager.setup();

  class EditorUIBuilder {
